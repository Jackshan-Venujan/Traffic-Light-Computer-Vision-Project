# LISA Traffic Light Detection with YOLOv8

End-to-end pipeline that takes the **LISA Traffic Light Dataset** (Supervisly-format
JSON annotations), converts it to YOLO format, trains a **YOLOv8** object detector,
runs inference on images and videos, and exports the trained model to **ONNX** for
deployment.

The entire workflow lives in a single Jupyter notebook
[`Traffic_Light_Complete_Workflow.ipynb`](Traffic_Light_Complete_Workflow.ipynb)
so it can be executed cell-by-cell or top-to-bottom.

## 2. Classes & Dataset

The LISA dataset's `meta.json` defines **14 source classes**, split into two groups:

| Group | Classes |
|---|---|
| **Bulb classes** (тЙИ2тАУ3 px at imgsz=640 тАФ *too small for YOLO*) | `go`, `go forward`, `go left`, `stop`, `stop left`, `warning`, `warning left` |
| **Housing classes** (тЙИ6тАУ13 px at imgsz=640 тАФ detectable) | `go traffic light`, `go forward traffic light`, `go left traffic light`, `stop traffic light`, `stop left traffic light`, `warning traffic light`, `warning left traffic light` |

The notebook **filters to the 7 housing classes** (step 7b) because YOLOv8's
smallest detector stride is ~8 px тАФ bulbs literally can't be learned. After
filtering, classes are renumbered `0тАУ6` and `data.yaml` is rewritten.

Final `data/processed/data.yaml`:
```yaml
path: data/processed
train: images/train
val: images/val
test: images/test

nc: 7
names:
  0: go forward traffic light
  1: go left traffic light
  2: go traffic light
  3: stop left traffic light
  4: stop traffic light
  5: warning left traffic light
  6: warning traffic light
```

## 6. Alternative: CLI Script

If you don't want to run the notebook, the converter step is also exposed as a
standalone script:

```bash
python scripts/prepare_dataset.py \
  --dataset-dir Dataset \
  --output-dir data/processed \
  --val-ratio 0.1
```

After it finishes, train directly with the Ultralytics CLI:
```bash
yolo detect train \
  data=data/processed/data.yaml \
  model=yolov8n.pt \
  epochs=50 \
  imgsz=1280 \
  batch=8 \
  project=runs \
  name=traffic_light_v2
```

(You will still need to run the **7-class filter** logic from notebook cell 16
before training, or the model will collapse on bulb labels.)

## 9. Tuning & Troubleshooting

| Symptom | Fix |
|---|---|
| `box_loss` stays at **0** during training | You're training on the un-filtered 14-class data тАФ bulbs are too small. Run cell 16 (the 7-class filter) and retrain. |
| **CUDA OOM** | Lower `batch` (8 тЖТ 4 тЖТ 2), or lower `imgsz` to 960. |
| **No detections** on a video | Lower `conf` (0.25 тЖТ 0.15 тЖТ 0.1). |
| **Tons of false positives** | Raise `conf` to 0.4+. |
| **Small / distant lights missed** | Use `imgsz=1280` (or 1536) at inference. |
| **Inference too slow** | `vid_stride=2` or `3` to skip frames; or use the ONNX export with ONNX Runtime. |
| **No GPU** | `device='cpu'` everywhere; expect training to take many hours. |
| Want **higher accuracy** | Swap `yolov8n.pt` тЖТ `yolov8s.pt` or `yolov8m.pt`, increase `epochs`. |

## 10. Why We Filter to 7 Classes

The first training runs (`runs/traffic_light_v1*`) collapsed: **all loss terms
stayed at 0**. Root cause: LISA labels both the **traffic-light housings**
(the boxes themselves, ~6тАУ13 px wide at `imgsz=640`) **and** the **individual
bulbs** (`go`, `stop`, `warning`, тАж тАФ only ~2тАУ3 px wide). YOLOv8's smallest
detection stride is ~8 px, so the bulb labels are below the network's
representational floor and poison training.

The fix (notebook cell 16):
1. Keep only classes whose name ends with `" traffic light"` (the housings).
2. Renumber them 0тАУ6.
3. Rewrite every label file in `data/processed/labels/`.
4. Rewrite `data.yaml` with `nc: 7`.

Subsequent runs (`runs/traffic_light_v2*`) trained successfully at
`imgsz=1280, epochs=50`.
